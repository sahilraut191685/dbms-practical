Improve Query Performance using Indexing, Consider a collection users: (Id, name,
mailed, age, city)
1. Create an index on the email field to improve search performance.
2. Create a compound index on city and age.
3. Query users from Mumbai aged above 20 efficiently.

```sql
-- COLLECTION: users (Id, name, email, age, city)

-- 1. CREATE INDEX ON EMAIL
CREATE INDEX idx_email ON users(email);

-- 2. CREATE COMPOUND INDEX ON CITY AND AGE
CREATE INDEX idx_city_age ON users(city, age);

-- 3. QUERY USERS FROM MUMBAI AGED ABOVE 20
SELECT *
FROM users
WHERE city = 'Mumbai' AND age > 20;
```





Create a database for a Student Course Management System.
 Create the following tables with appropriate attributes and constraints:
o Students
o Courses
o Enrollments
 Insert at least 5 records in each table.
Write SQL queries to:
1 Display student names, course names, and marks using INNER JOIN.
2 Display all students including those not enrolled in any course using LEFT JOIN.
3 Display all courses including those with no enrolled students using RIGHT JOIN.
4 Display all possible student–course combinations including unmatched records using FULL
OUTER JOIN (or equivalent).
5 Create a view named Student_Course_View to display Student Name, Course Name, and
Marks.
6 Create another view to display students who have scored more than 75 marks.
7 Retrieve data from both created views.
8 Drop any one of the created views.


```sql
-- CREATE TABLES
CREATE TABLE Students (
    student_id INT PRIMARY KEY,
    name VARCHAR(50) NOT NULL
);

CREATE TABLE Courses (
    course_id INT PRIMARY KEY,
    course_name VARCHAR(50) NOT NULL
);

CREATE TABLE Enrollments (
    enroll_id INT PRIMARY KEY,
    student_id INT,
    course_id INT,
    marks INT,
    FOREIGN KEY (student_id) REFERENCES Students(student_id),
    FOREIGN KEY (course_id) REFERENCES Courses(course_id)
);
```
